# Tutoring Applications Narrative

## Problem

Tutoring workflows are fragmented across messaging, scheduling, and administrative tracking.

## My Role

I contributed as a full-stack engineer across web, mobile, and backend components.

## Constraints

- Multi-client experience (web and mobile)
- Scheduling integrity and data consistency
- Need for admin visibility into session activity

## Key Decisions

1. Shared backend for consistent business logic
2. Structured relational schema for sessions and availability
3. Feature set centered on real tutor-student workflows

## Failure and Lesson

Initial scheduling logic did not handle all booking edge cases cleanly.
I refined constraints and data checks.
Lesson: availability systems require explicit edge-case handling from the start.

## Outcomes

- Unified platform for communication and booking
- Better coordination between students, tutors, and administrators
- Experience designing full-stack systems across mobile and web surfaces
